# Station Board

A self-contained, browser-based train arrivals board for any Amtrak station. No server, no build step, no dependencies to install — just open `index.html` in a browser.

The app learns train schedules automatically by watching the [Amtraker API](https://amtraker.com). Point it at any station code, and over a few days it builds a complete picture of every train serving that station.

![Dark themed arrivals board with amber text](https://img.shields.io/badge/style-dark%20amber-0a0c0f?style=flat-square&labelColor=0a0c0f&color=f5a623)

## Features

- **Self-learning schedule** — discovers trains automatically over time, persisted in `localStorage`
- **Works for any Amtrak station** — switch stations with the built-in station selector
- **Live train tracking** — real-time status (on time / late / early), estimated arrivals, nearest station
- **Satellite map** — click MAP on any active train to see its live position relative to the station (Esri World Imagery + Leaflet)
- **Proximity alert** — trains arriving within 10 minutes get a pulsing amber highlight
- **Railcam link** — when a train is arriving soon, a link to a YouTube railcam search appears automatically
- **Expandable rows** — click any active train to see its full station-by-station itinerary with schedule deltas
- **Auto-refresh** — fetches new data every 10 minutes with a visible countdown and progress bar
- **Dark aesthetic** — amber-on-black departure board look with scanline overlay, color-coded status badges

## Quick Start

**Option A — Open directly:**

Double-click `index.html` or open it in any modern browser. That's it.

**Option B — Serve on your local network:**

```bash
cd station-board
python3 -m http.server 8080
```

Then visit `http://<your-ip>:8080` from any device on the same network.

## Changing Stations

Type any Amtrak station code (e.g. `CHI`, `NYP`, `LAX`, `EMY`) into the station input field and click **Go**. The board resets and begins learning the new station's schedule from scratch. Previously learned schedules are preserved in `localStorage` — switching back restores them instantly.

A directory of Amtrak station codes is available at [amtrak.com](https://www.amtrak.com/stations).

## How the Self-Learning Schedule Works

The board starts empty. Each time the API returns data about a train stopping at the station, the app records:

- Scheduled arrival/departure times
- Route name and direction
- Origin and destination

This data is saved to `localStorage` per station. Trains that aren't currently active still appear on the board using their learned schedule. Over a few days of the page being open, the board fills in with every train that serves the station — no manual configuration needed.

Scheduled times are updated on every sighting, so the board stays current if Amtrak changes timetables.

## Tech Stack

- **HTML/CSS/JS** — single file, no framework, no build step
- **[Amtraker API](https://amtraker.com)** — live Amtrak train data (no API key required)
- **[Leaflet](https://leafletjs.com)** — interactive maps (loaded from CDN)
- **[Esri World Imagery](https://www.arcgis.com/)** — satellite map tiles with boundary labels
- **[Piped API](https://github.com/TeamPiped/Piped)** — YouTube search for railcam links (no API key required)
- **Google Fonts** — Bebas Neue, Share Tech Mono, DM Sans
- **localStorage** — schedule persistence across sessions
